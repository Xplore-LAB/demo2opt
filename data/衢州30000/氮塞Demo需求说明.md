# 氮塞 Demo 需求说明

> 来源：2026-04-27 会议文字记录。  
> 整理原则：尊重原始会议记录，把“会议原文依据”和“执行扩展”分开写。  
> 说明：本文件只整理 Demo 的扫描方式、窗口长度、界面形式和诊断输出，不把后续产品化设计当成老师原话。

---

## 1. 任务目标

建立一个用于展示的氮塞监测 Demo，用离线历史数据模拟在线监测过程。Demo 的重点不只是显示曲线，而是体现完整流程：

```text
窗口扫描
→ 氮塞检测
→ 异常触发
→ 溯源分析
→ 证据展示
→ 操作建议
```

---

## 2. 会议原文依据

### 2.1 扫描方式：事件触发还是定时扫描

会议一开始老师问：

```text
你是事件触发的扫描还是定时的扫描？
```

后续老师的原意是：

- 监测可以有两种方式；
- 事件触发可以；
- 固定周期扫描也可以；
- 在线检测本身也可以理解为固定周期不断看一下；
- 当前只是 Demo，不必在这个阶段把最终生产形态完全定死。

可确认结论：

| 项目 | 会议结论 |
|---|---|
| 事件触发 | 可以 |
| 固定周期扫描 | 可以 |
| Demo 阶段 | 可用离线数据模拟在线扫描 |
| 当前关键 | 不是触发模式本身，而是检测后能输出什么 |

---

### 2.2 窗口长度

老师明确表达：

```text
一分钟肯定不行。
稳态总得取 10 分钟以上。
半小时左右更稳。
窗口长度不能太短。
```

同时老师举了一个在线运行的例子：

```text
窗长可以长一点，比如半小时；
每隔 5 分钟滑动一下。
```

Demo 阶段老师也说：

```text
只要 5 分钟以上就行；
取 15 分钟或者 10 分钟都可以。
```

可确认结论：

| 参数 | 会议结论 |
|---|---|
| 1 min | 不合适，太短 |
| 5 min | Demo 可以作为下限讨论 |
| 10 min | 可选 |
| 15 min | 可选 |
| 30 min | 更接近稳态判断，老师举过例子 |
| 滑动步长 | 老师举例每 5 min 滑动一次 |

---

### 2.3 离线数据如何模拟在线扫描

老师原意：

- 你有一段离线数据；
- 可以从历史数据起点开始；
- 一次处理半小时数据；
- 下一次把窗口往前滚；
- 这就是模拟时间变化；
- 滚动频率由你定，因为只是离线模拟。

近原文整理：

```text
从昨天到今天的数据，可以从昨天的起点开始，半小时半小时滚动一次。
算完马上滚动，还是等多少时间再滚动，由你定。
离线只是模拟，这个没有关系。
```

---

### 2.4 DCS 风格与实时曲线

你问 Demo 是否要做成 DCS 样式，是否要把实时数据显示成曲线。

老师没有否定 DCS 风格，但明确把重点转到：

```text
后面出了建议以后，现在能出哪些信息？
后面的整个工作流怎么规划？
检测到了以后，溯源判断的逻辑清不清楚？
```

可确认结论：

```text
DCS 风格可以做，但当前最关键的是诊断输出和后续工作流。
```

## 4. 执行扩展：推荐 Demo 方案

> 以下是执行扩展，用于推进开发，不代表会议原文已全部定稿。

### 4.1 推荐扫描配置

```text
模式：离线历史数据模拟在线滚动扫描
窗口长度：默认 30 min
滑动步长：默认 5 min
备用窗口：10 min / 15 min
触发方式：每个窗口先检测，异常后进入溯源
```

### 4.2 设计理由

| 配置 | 理由 |
|---|---|
| 默认 30 min | 老师认为半小时更接近稳态判断 |
| 5 min 滑动 | 老师举例“半小时窗口，每 5 分钟滑动一下” |
| 支持 10/15 min | 老师说 Demo 可取 10 或 15 min |
| 不设 1 min | 老师明确说 1 min 太快，容易误判 |

## 5. 功能需求

### 5.1 数据导入

| 功能 | 要求 |
|---|---|
| 导入历史数据 | 支持 Excel / CSV |
| 时间字段识别 | 自动识别时间列并排序 |
| 测点字段识别 | 支持 AI705、AI701、FI702、FIQC701 等字段 |
| 数据质量检查 | 检查缺失值、重复时间、时间间隔异常 |

### 5.2 滚动窗口

| 功能 | 要求 |
|---|---|
| 窗口长度 | 默认 30 min，可选 10/15/30 min |
| 滑动步长 | 默认 5 min |
| 当前窗口显示 | 显示窗口起止时间 |
| 自动滚动 | 支持历史数据快速回放 |
| 单窗口查看 | 支持暂停查看某个窗口结果 |

### 5.3 氮塞检测输出

| 输出字段 | 说明 |
|---|---|
| 是否疑似氮塞 | 是 / 否 / 需人工确认 |
| 触发测点 | 哪些变量触发异常 |
| 异常程度 | 轻微 / 中等 / 严重，占位待量化 |
| 当前窗口 | 起止时间 |
| 是否进入溯源 | 异常时进入故障树分析 |

### 5.4 溯源输出

| 输出字段 | 说明 |
|---|---|
| 候选原因 | 多个可能原因 |
| 所属分支 | 粗氩 / 主塔 / 前端 / 操作 |
| 证据测点 | 支撑该原因的变量 |
| 判断依据 | 正常范围、趋势、偏差、人工确认项 |
| 可信度 | 先占位，后续设计评分 |
| 操作建议 | 建议动作 |
| 风险提示 | 调整过快、误判等风险 |

## 6. 页面结构建议

> 以下为执行扩展。

| 页面区块 | 内容 |
|---|---|
| 顶部状态栏 | 当前装置、当前时间、扫描状态 |

| 证据表区 | 测点当前值、历史范围、偏离程度 |
| 操作建议区 | 建议、风险、人工确认 |
| 历史记录区 | 每个窗口诊断结果列表 |

## 7. 给 AI IDE 的开发提示词草案

```text
请在当前氮塞监测 Demo 中增加“滑动窗口扫描 + 诊断输出”能力。

要求：
1. 支持导入历史 Excel/CSV 数据，按时间列排序；
2. 支持设置窗口长度，默认 30 min，可选 10 min、15 min；
3. 支持设置滑动步长，默认 5 min；
4. 对每个窗口计算关键变量统计特征；
5. 每个窗口输出是否疑似氮塞、异常等级和触发测点；
6. 如果疑似氮塞，则进入溯源分析区；
7. 溯源区暂时按粗氩系统、主塔、前端/分子筛、操作调节四类原因展示；
8. 每个候选原因必须显示证据测点、判断依据、可信度占位和人工确认标记；
9. 页面风格保持工业 DCS 风格，但不要只做视觉美化，重点是诊断结果结构清晰。
```
