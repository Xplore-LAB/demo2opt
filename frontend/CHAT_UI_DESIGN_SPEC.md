# 对话界面设计规范

基准参考：`frontend/src/views/前端设计参考图.png`

目标：让当前对话模块在未来嵌入整张大界面时，视觉语言、组件气质和交互节奏保持一致，同时保留本项目已有的分析流、实时监控和报告联动特性。

## 1. 视觉方向

- 整体气质：工业场景下的轻量智能工作台，保持克制、清洁、偏专业，不做消费级炫技。
- 布局骨架：左侧信息导航 + 中间对话主区 + 右侧工作面板，模块边界清晰。
- 背景策略：浅灰白底叠加极淡蓝色径向光斑，避免纯白平铺造成割裂。

## 2. 色彩系统

- 主色：`#2E6AA7`
- 主色强化：`#1F5C97`
- 主色浅底：`#DCEAFA`
- 页面底色：`#F3F4F6`
- 卡片底色：`rgba(255,255,255,0.88)` / `#FFFFFF`
- 常规边框：`rgba(145,163,186,0.26)`
- 强调描边：`#2F6FB1`
- 主文本：`#1F2A37`
- 次文本：`#697586`
- 成功：`#188567`
- 警告：`#B7791F`
- 危险：`#D64545`

规则：
- 主色只用于当前激活项、主按钮、焦点态和关键状态。
- 大面积区域优先使用浅色背景，避免整屏高饱和。
- 异常、警告、成功只作为状态信息点缀，不参与主导航配色。

## 3. 字体层级

- 页面主标题：30px / 800 / 紧凑字距
- 面板标题：30px 或 24px / 800
- 指标数字：30px / 800
- 卡片标题：16px-18px / 800
- 正文：14px / 1.7-1.8
- 辅助说明：12px-13px / 1.6
- 标签与状态胶囊：12px / 700

规则：
- 标题用更重字重拉开层级，不依赖高饱和色。
- 次级文案统一落在 muted 色，不和正文争夺注意力。

## 4. 间距与圆角

- 页面外层间距：24px-32px
- 模块内边距：18px-24px
- 组件间距：8px / 12px / 16px / 20px 四档
- 大卡片圆角：24px
- 常规卡片圆角：18px
- 按钮和输入框圆角：14px-16px
- 对话气泡圆角：22px

规则：
- 统一使用偏大圆角，贴近参考图的“模块化卡片”观感。
- 同一层级模块使用同一圆角，不混用 8px、12px、20px 等杂乱值。

## 5. 边框与阴影

- 默认边框：1px 浅灰蓝
- 强调卡片：顶部细色条或浅蓝底，不直接用厚描边
- 阴影：低对比、大扩散、短透明度

推荐阴影：
- 模块阴影：`0 18px 40px rgba(41,72,112,0.08)`
- 卡片阴影：`0 14px 30px rgba(28,53,91,0.08)`

## 6. 组件规范

- 侧栏卡片：白底或浅蓝渐变底，顶部可带细强调条。
- 标签页：胶囊按钮，激活态使用浅蓝背景 + 主色文字。
- 输入框：浅色底、细边框、16px 圆角，焦点态用浅蓝外发光。
- 主按钮：蓝色渐变底，字重 700，保留明确可点击感。
- 次按钮：浅蓝底或白底描边，不和主按钮抢主导权。
- 对话气泡：AI 气泡偏白，用户气泡偏浅蓝，状态气泡按成功/警告/异常轻度着色。
- KPI 卡片：白底带微渐变，数字权重大于说明文案。

## 7. 动效规范

- 动效时长：180ms-240ms
- 缓动：`cubic-bezier(.2,.8,.2,1)`
- 使用场景：卡片浮起、按钮 hover、消息淡入上移、焦点态
- 禁止：大幅位移、连续弹跳、复杂旋转

规则：
- 动效用于“增强反馈”，不是“制造存在感”。
- 需要兼容 `prefers-reduced-motion`。

## 8. 本项目保留特征

- 保留实时状态、阶段推进、人工确认、监控快照、报告导出这些分析型工作流能力。
- 保留左右双工作区结构，不把页面改成纯聊天产品。
- 对话区仍承担“分析过程可视化”的职责，不只展示普通问答文本。

## 9. 落地约束

- 优先修改样式层和少量结构 class，不改动业务逻辑。
- Element Plus 组件统一做皮肤覆盖，避免默认主题破坏整体一致性。
- 移动端保持单列降级，不牺牲现有功能入口。
