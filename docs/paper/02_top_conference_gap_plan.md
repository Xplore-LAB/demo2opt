# 顶会差距清单与冲刺计划

## 当前定位（实话实说）

- 当前项目属于“强工程系统 + 应用案例”。  
- 若冲顶会主会（如 NeurIPS/ICLR/ICML 主轨），仍缺：  
  - 方法创新可独立命名  
  - 研究级实验强度  
  - 理论或可验证机制  

## 差距清单（可验收）

### 1) 问题定义与方法抽象不足

- 要求：从工程流程升级为形式化问题（状态、动作、检索函数、约束函数）。  
- 验收：方法章包含符号表、目标函数、约束定义。  

### 2) 创新点不够“可发表”

- 要求：至少 1 个可命名的方法创新（不是模块拼接）。  
- 验收：给出算法名、伪代码、与 SOTA 差异点。  

### 3) 基线与实验深度不足

- 要求：强基线、消融、鲁棒性、显著性检验完整。  
- 验收：主表 >= 5 基线，消融 >= 3 组，给出 p-value。  

### 4) 安全与可解释性证据不足

- 要求：把风险门控和解释输出做定量评估。  
- 验收：安全违规率、专家一致性、建议可执行率等指标齐全。  

### 5) 复现链路不足

- 要求：一键复现实验、图表重建脚本、固定随机种子。  
- 验收：新同学按文档可在 1 天内重建主实验结果。  

## 12 周冲刺计划（建议）

### 第 1-2 周：方法冻结

- 完成任务形式化、核心算法草案、贡献点定稿。  
- 输出：方法定义文档 + 算法伪代码。  

### 第 3-4 周：实验协议冻结

- 固定数据划分、指标定义、基线清单、统计检验方法。  
- 输出：实验协议 v1。  

### 第 5-6 周：主方法与基线实现

- 完成统一训练/推理/评测脚本。  
- 输出：可运行 baseline + full method。  

### 第 7-8 周：主实验与消融

- 跑主表与消融，整理置信区间与显著性。  
- 输出：Table 1 / Table 2 初版。  

### 第 9-10 周：鲁棒性与失败分析

- 噪声、缺失、漂移、超时等场景测试。  
- 输出：Table 3 + 失败案例图。  

### 第 11 周：论文初稿

- 完成全文初稿与图表草版。  

### 第 12 周：审稿人视角打磨

- 对照拒稿风险逐条修复，完成投稿版。  

## 拒稿高风险点（提前规避）

1. “只是系统集成，没有方法创新”。  
2. “只做了单场景案例，缺泛化”。  
3. “结论无统计显著性支撑”。  
4. “复现信息不足”。
