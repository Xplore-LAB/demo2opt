## 1. 发明名称
**一种基于多维核心指标耦合机理与模糊语义推理的空分装置运行优化方法及系统**  
*(Method and System for Air Separation Unit Operation Optimization Based on Multi-dimensional Core Indicator Coupling Mechanism and Fuzzy Semantic Reasoning)*

## 2. 技术领域
本发明涉及工业过程控制与人工智能技术领域，特别是涉及一种针对大型空分装置（ASU），通过提取率、稳定性和单耗三大核心指标的耦合分析，利用模糊逻辑（Fuzzy Logic）进行语义量化，并结合大语言模型（LLM）进行思维链（CoT）推理的智能运行优化决策技术。

## 3. 背景技术与现有缺陷
现有空分装置运行优化主要存在以下痛点：
1.  **指标评价孤立**：通常仅关注单一指标（如氧纯度），缺乏对**提取率**、**稳定性**和**单耗**三大核心指标的综合权衡，导致“按下葫芦浮起瓢”（例如：过度追求提取率导致稳定性恶化）。
2.  **数值语义鸿沟**：传统 DCS 系统仅提供冷冰冰的数值（如 `T=98.5`），缺乏对“优劣程度”的量化描述（如“良好”、“偏差较大”），大模型直接处理数值容易产生幻觉。
3.  **推理过程不可解释**：纯数据驱动的 AI 优化往往是“黑盒”，缺乏清晰的逻辑链条（如“指标 -> 影响因素 -> 优化手段”），操作员难以信任。

## 4. 本发明解决的技术问题
1.  **多指标耦合权衡**：建立三大核心指标的耦合分析机制，避免局部优化导致的整体性能下降。
2.  **数值语义化**：引入隶属度函数，将连续的传感器数值映射为标准化的语义状态，为大模型推理提供可靠输入。
3.  **可解释的决策路径**：构建“指标计算 -> 知识检索 -> 耦合推理”的三层逻辑闭环，生成具备思维链（CoT）的优化建议。

## 6. 核心方法步骤（权利要求主干）

本发明提供一种空分装置运行优化方法，包括以下步骤：

**S1：核心指标体系构建与实时计算**
*   定义三大核心指标集 $I = \{I_{ext}, I_{stab}, I_{ec}\}$：
    *   $I_{ext}$（提取率）：计算氧提取率 $\eta_{O2}$ 与氩提取率 $\eta_{Ar}$。
    *   $I_{stab}$（稳定性）：统计时间窗口 $T_w$ 内的氮塞发生次数 $N_{plug}$ 及产品纯度合格率 $R_{purity}$。
    *   $I_{ec}$（单耗）：计算单位氧气产量电耗 $E_{unit}$。
*   *(代码支撑：`src/services/asu_pipeline.py`)*

**S2：基于隶属度的语义量化评价**
*   为每个核心指标 $I_k$ 构建隶属度函数 $\mu_k(x)$：
    *   以**设计值** $D_k$ 为中心，**历史运行均值** $\bar{x}_k$ 为参考，确定最优区间 $[O_{min}, O_{max}]$。
    *   计算当前值 $x_k$ 的隶属度 $\alpha_k = \mu_k(x_k) \in [0, 1]$。
        *   若 $x_k$ 落在最优区间内，则 $\alpha_k \to 1.0$（状态：优秀）。
        *   若 $x_k$ 偏离最优区间但未超限，则 $\alpha_k \in (0.5, 0.9)$（状态：良好/一般）。
        *   若 $x_k$ 超出安全边界，则 $\alpha_k \to 0.0$（状态：严重异常）。
*   生成标准化语义向量 $V_{sem} = \{ (k, \alpha_k, S_k) | k \in I \}$，其中 $S_k$ 为语义描述。
*   *(代码支撑：`src/services/data_semantics.py`)*

**S3：瓶颈识别与知识库检索（逻辑第一层 -> 第二层）**
*   根据语义向量 $V_{sem}$，识别隶属度 $\alpha_k$ 最低（或低于阈值 $\theta$）的指标作为**瓶颈指标** $I_{bottle}$。
*   以 $I_{bottle}$ 为索引键，检索**运行优化知识库**，获取候选优化手段集合 $M = \{m_1, m_2, ..., m_n\}$（如：$m_1$=增加膨胀机制冷量, $m_2$=调整下塔回流比）。
*   *(代码支撑：`src/services/knowledge_base.py`)*

**S4：多维耦合推理与方案优选（逻辑第二层 -> 第三层）**
*   构建包含“当前语义向量 $V_{sem}$”、“候选手段 $M$”及“数据趋势特征”的**结构化思维链（CoT）Prompt**。
*   输入大语言模型（LLM），指示模型执行以下推理步骤：
    1.  **可行性评估**：结合当前测点数据（如阀门开度、压力余量），判断手段 $m_j$ 是否可执行。
    2.  **耦合效应推演**：预测执行 $m_j$ 后，对非瓶颈指标的**正向/负向影响**（如：$m_j$ 提升提取率 $I_{ext}$ 但可能导致稳定性 $I_{stab}$ 下降）。
    3.  **综合权衡**：基于预设的优先级权重（如稳定性 > 提取率 > 单耗），筛选最优优化方案 $P_{opt}$。
*   *(代码支撑：`src/services/reasoning_engine_v2.py`)*

**S5：闭环验证与策略输出**
*   输出优化方案 $P_{opt}$，包含具体操作步骤（Actionable Steps）及预期指标变化。
*   结合仿真模型（Simulation Model）对 $P_{opt}$ 进行验证，若预测风险过高则回退至 S4 重新筛选。
*   *(代码支撑：`src/services/decision_service.py`)*

## 7. 具体实施方式（结合代码实现）

### 7.1 隶属度函数设计
在 `src/services/data_semantics.py` 中，针对不同类型的指标设计不同的隶属度函数：
*   **越低越好型（如单耗）**：采用 Z 型隶属度函数。
    ```python
    def z_membership(x, min_val, max_val):
        if x <= min_val: return 1.0
        if x >= max_val: return 0.0
        return (max_val - x) / (max_val - min_val)
    ```
*   **越高越好型（如提取率）**：采用 S 型隶属度函数。
*   **中间最优型（如纯度）**：采用梯形或高斯隶属度函数。

### 7.2 知识库构建与检索
在 `src/services/knowledge_base.py` 中：
*   **解析**：利用 `MinerU` 解析非结构化 PDF 文档（操作手册、事故案例）。
*   **索引**：利用 `FAISS` 建立向量索引，支持基于语义的模糊检索。
*   **检索**：输入瓶颈指标描述（如“氧提取率偏低”），返回 Top-K 个相关的优化操作条目。

### 7.3 思维链（CoT）Prompt 模板
在 `src/services/reasoning_engine_v2.py` 中，构建如下 Prompt：
```text
【当前状态】
- 瓶颈指标：氧提取率 (隶属度 0.4, 严重偏低)
- 关联指标：氮塞频率 (隶属度 0.9, 优秀)

【候选手段】
1. 增加膨胀机制冷量
2. 提高上塔回流比

【推理任务】
请分析上述手段的可行性及其对“稳定性”和“单耗”的耦合影响。
思考步骤：
1. 手段 1 是否可行？（检查膨胀机当前负荷）
2. 手段 1 对提取率有何影响？对稳定性有何副作用？
3. ...
4. 综合推荐最优方案。
```
