# 精简版目录重构方案

## 核心原则
- **最小化改动**：只修复明显的问题
- **保持兼容**：不破坏现有功能
- **渐进式**：可以分步执行

## 需要立即修复的问题

### 1. 前端中文目录（影响跨平台）
```
frontend/参考的layouts/  →  删除（已有 layouts/）
```

### 2. 资源文件位置（命名不规范）
```
src/resets/  →  resources/
```

### 3. 根目录临时文件
```
main-run.pid  →  添加到 .gitignore
llm_configs.json  →  移到 config/
```

## 建议但非必需的改进

### 4. 文档整理
```
docs/
├── api/           # API 文档（合并 specs/）
├── guides/        # 用户指南
├── research/      # 研究文档（保持）
└── development/   # 开发文档（合并其他）
```

### 5. 脚本分类
```
scripts/
├── run_interactive.py      # 主要脚本保持根级
├── start_web.py
├── dev/                    # 开发工具
└── experiments/            # 实验脚本
```

## 不建议改动的部分

✅ **保持不变**：
- `src/` 源码结构（已经很好）
- `tests/` 测试结构（功能正常）
- `frontend/src/` 前端源码
- `config/` 配置文件
- `data/` 数据文件

## 执行计划

**阶段 1**（安全，立即执行）：
1. 删除前端中文参考目录
2. 移动资源文件到 resources/
3. 更新 .gitignore

**阶段 2**（可选，后续执行）：
1. 文档重组
2. 脚本分类

## 风险评估
- ✅ 阶段1：零风险（不影响代码）
- ⚠️ 阶段2：低风险（需要更新文档链接）
