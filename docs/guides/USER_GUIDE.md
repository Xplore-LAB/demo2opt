# 空分装置智能优化系统 - 使用说明书

## 一、系统架构

当前系统采用**多进程架构**：

```
┌─────────────────────────────────────────────────────────┐
│                      你的电脑                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────────────────────┐    ┌─────────────────────┐   │
│   │   HTTP 服务器       │    │  WebSocket 服务器  │   │
│   │   (主进程)          │    │   (子进程)         │   │
│   │   端口 8888         │    │   端口 8889        │   │
│   │   提供网页界面       │    │   实时通信推送    │   │
│   └─────────┬───────────┘    └─────────┬───────────┘   │
│             │                          │               │
│             └──────────┬───────────────┘               │
│                        ▼                              │
│              ┌─────────────────┐                      │
│              │   浏览器       │                      │
│              │  http://localhost:8888 │             │
│              └─────────────────┘                      │
│                                                         │
│   ┌─────────────────────────────────────────────────┐ │
│   │   Flask REST API (可选，后台运行)               │ │
│   │   端口 5000 (或自定义)                         │ │
│   └─────────────────────────────────────────────────┘ │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**架构说明：**

| 组件 | 实现方式 | 说明 |
|------|---------|------|
| HTTP 服务器 | `socketserver.TCPServer` | 提供静态文件和前端页面 |
| WebSocket 服务器 | `websockets` 库 (独立进程) | 推送分析进度和实时日志 |
| REST API | Flask + CORS | 提供任务状态查询（可选） |
| CORS | HTTP 层 + Flask 层 | 允许跨域请求 |

**简单理解：**
- 只需要启动一个命令 `python start_web.py`
- 浏览器访问 `http://localhost:8888` 即可使用

## 二、快速开始

### 步骤 1：安装依赖

在项目目录下打开终端，运行：

```bash
pip install -r requirements.txt
```

### 步骤 2：启动系统

```bash
python start_web.py
```

启动成功后会显示：
```
================================================
工业空分装置智能优化系统 v2.0 - Web 界面
================================================

HTTP 服务器已启动: http://localhost:8888
访问 http://localhost:8888/index.html 查看界面
WebSocket 服务器已启动
```

### 步骤 3：使用浏览器访问

打开浏览器，访问：**http://localhost:8888**

你会看到空分装置智能优化系统的界面。

## 三、使用流程

### 1. 配置 API（可选）

界面顶部有 **API 配置** 面板：

- **Dify 模式**：如果你有 Dify API，填写 URL 和 Key
- **直接 LLM 模式**：如果你有 OpenAI/DeepSeek 等 API，填写 URL、Key 和模型

> ⚠️ **安全提示**：API Key 存储在前端浏览器中，仅本地运行时不暴露。若部署到公网，建议修改代码将 Key 移到后端环境变量。

> 💡 **提示**：如果不配置，系统会以 Mock 模式运行，使用模拟数据进行演示。

### 2. 上传数据

有两种方式：

**方式 A：上传文件**
- 点击上传区域，选择 CSV 或 Excel 文件
- 支持 `.csv`, `.xlsx`, `.xls` 格式

> 📋 **Excel 文件格式要求**：
> - 第一列：时间戳
> - 第二列起：传感器数据（第一行为描述，第二行为设计值，后续为数值）

**方式 B：使用示例数据**
- 点击「使用示例数据」按钮
- 系统会加载内置的演示数据

### 3. 开始分析

点击「开始分析」按钮，系统会：

1. **数据加载** - 解析 Excel/CSV 文件，执行基础数据质量检查
2. **语义分析** - 计算衍生指标，进行异常检测和状态评估
3. **知识检索** - 从本地知识库检索相关案例和解决方案
4. **五维推理** - 调用 LLM 进行深度分析：
   - 宏观工况定位
   - 核心产出指标评估
   - 精馏工艺参数分析
   - 能量与冷量平衡分析
   - 优化建议路径生成
5. **决策建议** - 生成操作指导和风险预警
6. **报告生成** - 输出 Markdown 和 PDF 格式报告

### 4. 查看结果

分析完成后，界面会显示：

- **异常指标**：哪些指标超标/异常
- **五维深度推理**：详细的分析推理过程
- **决策建议**：优化建议和操作指导
- **下载报告**：可以导出 Markdown 格式报告

> 💡 **注意**：当前 WebSocket 仅推送**阶段性日志和进度**，不支持 LLM Token 流式输出。LLM 响应会一次性返回后通过 WebSocket 推送。

## 四、系统能力边界

### 支持的数据规模

| 指标 | 限制 | 说明 |
|------|------|------|
| 单文件行数 | ~10,000 行 | 取决于 Excel 内存占用 |
| 传感器数量 | ~100 个 | 单 Sheet 建议不超过此数量 |
| 文件格式 | .xlsx, .xls, .csv | UTF-8 编码优先 |

> ⚠️ **大规模数据注意**：当前为单机演示版本，不支持百万级时间序列。若需处理大规模数据，建议：
> - 使用数据分片加载
> - 引入时序数据库
> - 迁移到服务端架构

### 数据质量检查

系统会自动执行以下检查：

1. **缺失率统计** - 报告空值比例超过 0% 的列
2. **常量序列识别** - 识别标准差为 0 的列（可能是故障传感器）
3. **异常值检测** - Z-score > 3 的数据点标记为异常

### LLM 调用配置

| 模式 | 超时 | 重试 | 降级策略 |
|------|------|------|----------|
| Dify API | 120s | 1次 | 失败后切换 Mock |
| Direct LLM | 120s | 1次 | 失败后切换 Mock |
| Mock 模式 | - | - | 返回预设分析结果 |

## 五、常见问题

### Q1: 启动报错 "端口被占用"

```bash
# Windows 查看占用 8888 端口的程序
netstat -ano | findstr :8888

# 结束占用进程
taskkill /PID <进程ID> /F
```

### Q2: 浏览器显示空白

1. 确认 `python start_web.py` 正在运行
2. 确认没有其他程序占用 8888 端口
3. 尝试刷新浏览器（Ctrl+F5）
4. 打开浏览器开发者工具 (F12) 查看 Console 错误

### Q3: WebSocket 连接失败

1. 检查后端是否正常启动，确认显示 "WebSocket 服务器已启动"
2. 检查浏览器 Console 是否有跨域错误
3. 确认使用的是 `localhost` 而非 IP 地址

### Q4: 分析一直转圈

1. 检查 API 配置是否正确
2. 查看 Console 日志中的错误信息
3. 或者使用 Mock 模式（不配置 API）

### Q5: 数据加载失败

1. 检查 Excel 文件格式是否符合要求
2. 确认文件编码为 UTF-8
3. 查看 Console 中的详细错误信息

### Q6: LLM API 调用失败

1. 确认 API Key 正确且未过期
2. 检查网络连接
3. 确认 API URL 可访问
4. 系统会自动降级到 Mock 模式

## 六、文件说明

## 七、开发相关

### 启动开发模式（修改前端代码时）

```bash
# 终端1：启动后端
python start_web.py

# 终端2：启动前端开发服务器
npm install
npm run dev
```

然后访问 **http://localhost:8080**

### 启动独立 REST API 服务

```bash
python api_server.py
```

服务地址：**http://localhost:5000**

可用接口：
- `GET /api/health` - 健康检查
- `GET /api/tasks` - 获取所有任务
- `GET /api/tasks/<task_id>` - 获取指定任务详情
- `GET /api/tasks/<task_id>/progress` - 获取任务进度

### 运行测试

```bash
python -m pytest tests/e2e/test_ui_flow.py -v -s
```

## 八、技术限制与已知问题

1. **LLM 不支持流式输出** - 当前为整段返回，未来可升级为 SSE
2. **单用户单任务** - 不支持多并发分析任务
3. **无权限管理** - 所有用户共享同一配置
4. **知识库需手动构建** - 需通过 scripts 导入历史案例
5. **报告格式单一** - 仅支持 Markdown 和 PDF，不支持 JSON/XML
6. **无历史任务查询** - 每次刷新页面状态丢失
