# 氮塞识别 Demo 返修方案：从固定阈值改为自适应工作点 + 图形形态识别

## 一、返修背景

当前 Demo 给人的感觉偏“程序阈值扫描”，老师主要关注点是：

> AI705 的正常工作点会随装置、仪表校表、工况和厂家差异发生变化，不能按 97、98、99.5 这类固定绝对值判断氮塞。  
> 应先看曲线是否存在“下凹—谷底—回升”的形态，再根据相对下凹幅度、持续时间和恢复情况判断轻微、中等或严重。

因此 Demo 的核心逻辑应从：

```text
AI705 低于某个固定值 → 标为疑似氮塞
```

调整为：

```text
识别当前工作点
    ↓
判断是否存在相对当前工作点的下凹形态
    ↓
识别开始、谷底、结束、多小谷、恢复
    ↓
根据相对下凹幅度和持续时间分级
    ↓
联看 AI701、FI702、FIC701、负荷变量进行原因分析
```

## 二、当前 Demo 主要问题

### 2.1 识别逻辑表达显得刚性

当前页面顶部虽然已经提到“自适应工作点”和“下凹回升形态”，但页面呈现还不够“看图识别”。当前页面给人的直观感受仍像：

1. 程序自动扫了一遍；
2. 标了几个红色区间；
3. 右侧输出静态摘要；
4. 原因分析按钮还没有真正被激活。

页面应更明确展示：

- 当前工作点是多少；
- 下凹是相对谁下凹；
- 下凹幅度是多少；
- 是否形成谷底；
- 是否回升；
- 是单谷还是多谷；
- 为什么是轻微 / 中等 / 严重。

---

### 2.2 红色区间标注过于单薄

当前图上只显示红色竖条，容易像“随便标了两个时间点”。建议改成事件级标注：

| 标注元素 | 作用 |
|---|---|
| 半透明红色背景 | 表示整个候选事件区间 |
| 起点竖线 | 标出下凹开始 |
| 谷底竖线 | 标出最低点 |
| 结束竖线 | 标出恢复或低位结束 |
| 自适应工作点线 | 显示正常工作点 |
| 正常波动带 | 显示正常波动范围 |
| 下凹深度箭头 | 显示当前下凹幅度 |
| 多小谷编号 | 标出一个事件内的多个小谷 |

---

### 2.3 事件卡片缺少“图形判断依据”

右侧卡片应从“候选区间列表”升级为“图形识别证据卡”。

建议每个事件卡片展示：

```text

形态：多谷下凹 / 单谷下凹 / 连续低位 / 短时波动
工作点：99.86
最低值：98.72

持续时间：65 min
恢复比例：82%
小谷数量：3
分级：中风险疑似
结论：形态成立，建议进入联动复核
```

## 三、更新后的识别逻辑

### 3.1 先算当前工作点

不能用全局固定值。应按窗口计算当前工作点。

推荐做法：

```text
1. 对 AI705 做 1 min 中位数重采样；
2. 对 AI705 做 5 min 滚动中位数平滑；
3. 在 6 h 或 24 h 窗口内估计正常工作点；
4. 对明显下凹点做剔除或降权；
5. 使用剔除下凹后的中位数 / 高分位稳定值作为当前工作点。
```

可选实现：

| 方法 | 说明 | 适用场景 |
|---|---|---|
| 滚动中位数 | 简单稳定 | 第一版 Demo |
| 滚动高分位线 | 更贴近正常上沿 | AI705 下凹较多时 |
| STL / LOESS 基线 | 更平滑 | 后续优化 |
| 多模态辅助确认工作点 | 让大模型看图判断正常平线位置 | 展示效果更好 |

推荐第一版采用：

```text
当前工作点 = 剔除候选下凹点后的 6 h 滚动中位数
```

---

### 3.2 再识别相对下凹形态

判断对象从“AI705 是否低于某个值”改为：

```text
当前工作点 - AI705平滑值
```

只要出现相对当前工作点的连续下凹，并形成明确形态，就进入候选。

推荐特征：

| 特征 | 含义 |
|---|---|
| dip_depth | 当前工作点 - 谷底值 |
| dip_duration | 下凹持续时间 |
| recovery_ratio | 恢复比例 |
| valley_count | 小谷数量 |
| dip_area | 下凹面积 |
| shape_type | 单谷 / 多谷 / 连续低位 / 噪声波动 |
| visual_confidence | 多模态看图置信度 |

---

### 3.3 轻微、中等、严重按“相对下凹程度”分级

初版分级建议如下，后续根据专家标注调整：

| 分级 | 相对下凹幅度 | 持续时间 | 形态要求 |
|---|---:|---:|---|
| 观察 | 0.20 ~ 0.30 | 短时 | 有波动但形态不完整 |
| 轻微疑似 | ≥ 0.30 或 ≥ 正常波动 3 倍 | ≥ 10 min | 有下凹形态 |
| 中风险疑似 | ≥ 0.80 | ≥ 30 min | 下凹、谷底、恢复较清楚 |
| 高风险疑似 | ≥ 2.00 | ≥ 60 min | 下凹深、持续长、恢复明显或多测点联动 |

说明：

1. 这里的 0.30、0.80、2.00 是“相对当前工作点的下凹幅度”，不是 AI705 的绝对值；
2. 工作点可为 99.5、99.7、99.8，分级逻辑不受固定绝对值影响；
3. 轻微氮塞可出现在 99.5 ~ 98 之间，只要有清晰下凹形态，就不能漏掉；
4. 严重程度由相对下凹幅度、持续时间、恢复情况和联动证据共同决定。

## 四、多模态模型在 Demo 中的角色

老师希望“用看图方式识别”。因此多模态模型应作为第一入口，但数据计算仍负责精确认定。

### 4.1 多模态负责

| 任务 | 输出 |
|---|---|
| 看 AI705 标准趋势图 | 是否存在下凹形态 |
| 判断形态类型 | 单谷、多谷、连续低位、短时波动 |
| 判断是否回升 | 完整恢复、部分恢复、未恢复 |
| 判断多个小峰是否属于同一事件 | 合并或拆分建议 |
| 给出视觉风险等级 | 观察、轻微、中等、高风险 |
| 提醒需要联看的测点 | AI701、FI702、FIC701、FIQ101 等 |

### 4.2 程序负责

| 任务 | 输出 |
|---|---|
| 精确开始时间 | start_time |
| 精确谷底时间 | valley_time |
| 精确结束时间 | end_time |
| 计算下凹幅度 | dip_depth |
| 计算持续时间 | duration_min |
| 计算恢复比例 | recovery_ratio |
| 计算前中后均值 | evidence_table |
| 生成诊断卡片 | diagnosis_card |

### 4.3 推荐表述

```text
多模态模型负责“像现场专家一样先看图判断形态”，程序负责“把图上判断落到精确时间和数值”。
```


## 七、更新后的演示口径

### 7.1 讲 E11

```text
这里不是按 AI705 小于 97 或 98 来判断，而是先识别当前 AI705 的正常工作点。系统发现 E11 事件中 AI705 相对工作点出现明显下凹，并形成清晰谷底和恢复过程，持续时间 115 分钟，下凹幅度达到高风险水平。同时 AI701、FI702、FIC701 在同一窗口内有联动变化，因此系统将其标为强疑似氮塞。
```

### 7.2 讲 E08

```text
E08 图上同样有明显下凹，所以系统会先识别为疑似。但进入联动复核后，发现原料空气量和产品负荷变化较大，因此系统不会直接判为典型氮塞，而是提示“疑似但需排除负荷扰动”。这说明系统既能识别形态，也能避免把所有下凹都直接判成氮塞。
```

### 7.3 讲 E15 / E26

```text
E15 和 E26 是小幅、短时下凹。系统不会因为它没有跌到 97 或 98 就漏掉，也不会直接判为严重氮塞，而是标记为观察或轻微疑似。后续如果出现 AI701、FI702、FIC701 联动，再提升风险等级。
```

## 八、最终结论

Demo 应从“绝对阈值识别”全面调整为“自适应工作点 + 下凹形态识别”。

第一版应明确展示：

1. AI705 当前工作点自动估计；
2. 相对工作点的下凹形态；
3. 下凹开始、谷底、恢复结束；
4. 单谷 / 多谷 / 连续低位等形态识别；
5. 轻微、中等、严重分级；
6. AI701、FI702、FIC701、FIQ101 等联动复核；
7. 对负荷扰动和边界样本的谨慎处理。

这样更符合现场专家的判断方式，也更能体现多模态大模型“看图识别”的价值。
