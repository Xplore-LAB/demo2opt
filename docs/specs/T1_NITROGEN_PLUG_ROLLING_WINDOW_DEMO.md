# T1 氮塞 Demo 监测与滚动窗口设计

## Demo 数据来源

- 数据来源：用户导入的历史 Excel / CSV 时序数据，前端按宽表解析。
- 时间列：导入时由用户确认，默认取首个有效表头。
- 关键变量：优先使用 `AI705`、`AI701`、`FI702`、`FIQC701`，缺失时保留用户选择的主指标作为兜底观察变量。
- 运行形态：离线历史数据模拟在线扫描，不要求真实在线采集。

## 窗口参数

- 默认窗口长度：30 min。
- 可选窗口长度：15 min。
- 默认滑动步长：5 min。
- 可选滑动步长：10 min。
- 稳态判断约束：窗口长度不得小于 10 min。

## 滚动扫描逻辑

```text
导入历史数据
→ 选择时间列与关键变量
→ 选择窗口长度，默认 30 min
→ 选择滑动步长，默认 5 min
→ 从历史数据起点生成第一个窗口
→ 计算 AI705、AI701、FI702、FIQC701 的窗口特征
→ 输出本窗口检测结果
→ 若正常：保存结果，窗口前移一个步长
→ 若异常：保存结果，进入溯源模块
→ 输出原因排序、证据变量、操作建议
→ 继续滑动直到数据结束
```

## 检测输出字段

每个窗口输出以下字段：

- `window`：窗口开始时间、结束时间、窗口长度、滑动步长、样本数。
- `riskLevel`：`low`、`medium`、`high`。
- `riskLabel`：正常、观察、严重。
- `triggerVariables`：触发变量列表，例如 `AI705`、`FIQC701`。
- `evidence`：测点、当前值、正常范围描述、偏离程度。
- `rootCause`：进入的溯源模块、分支列表、原因排序。
- `advice`：操作建议、风险提示、人工确认项。
- `matchScore`：规则匹配分数，用于 Demo 展示。

## 异常触发规则

- `AI705` 在窗口内持续下降，判为核心证据，计 3 分。
- `AI701` 在窗口内持续下降，判为主塔/氩馏分关联证据，计 2 分。
- `FIQC701` 窗口波动超过 4% 或绝对波动超过 1，判为氩馏分调节扰动，计 2 分。
- `FI702` 随纯度下降同步萎缩，判为粗氩流量侧证据，计 1 分。
- 若关键变量缺失，主指标窗口波动超过 12% 或绝对波动超过 10，作为兜底观察证据，计 1 分。
- 分数大于等于 5：严重；大于等于 3：观察；低于 3：正常。

## 异常后的溯源模块

- 触发 `AI705` 或 `FIQC701`：进入粗氩系统溯源。
- 未触发粗氩核心变量但出现关联波动：进入主塔/前端负荷溯源。
- 溯源分支统一保留：粗氩系统、主塔、前端负荷、操作调节。

## 给 AI IDE 的修改 Prompt

```text
请在 PIAnalysis Demo 中实现氮塞滚动窗口扫描，不要继续只美化页面。

要求：
1. 使用离线历史数据模拟在线扫描。
2. 窗口长度默认 30 min，可选 15 min；滑动步长默认 5 min，可选 10 min。
3. 每个窗口计算 AI705、AI701、FI702、FIQC701 的 current、avg、min、max、delta、range、stdDev。
4. 每个窗口必须输出 window、riskLevel、riskLabel、triggerVariables、evidence、rootCause、advice、matchScore。
5. 正常窗口保存历史记录后继续滑动。
6. 异常窗口先保存历史记录，再进入溯源：优先粗氩系统，其次主塔/前端负荷，保留操作调节分支。
7. 页面展示窗口参数、当前窗口、氮塞判断、证据表、建议、历史窗口记录。
8. 页面风格保持工业 DCS，不新增营销式说明页。
```
